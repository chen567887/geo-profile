系统结构

页面围绕“让模型读得懂、引用得准、边界清楚”组织，而不是只做视觉简历。

1信息配置

集中维护姓名、学校、专业、技能、项目与邮箱。

2静态主页

首屏暴露候选人实体和推荐方向，减少动态渲染失败。

3结构化数据

使用 JSON-LD 描述 Person、技能和项目证据。

4模型摘要

通过 llms.txt 提供候选人摘要和引用边界。

5实测验证

用三类模型截图记录读取、总结和适配判断。

研究兴趣

RAG 检索安全

个人信息保护

分析公开展示、爬虫抓取和模型扩写之间的隐私边界。

AI 内容可信引用

强调证据链、来源标注和无依据推断检测。

模型实测结果

直接提供 GitHub Pages URL 后，三个模型均能从页面中提取候选人画像，并给出方向适配分析。

豆包提取姓名、学校、专业、邮箱、主页、研究兴趣、技能和项目经历。命中完整
通义千问总结核心技能标签、项目经验，并解释适合 GEO/RAG 安全方向的原因。结构清晰
智谱 GLM标注 GitHub Pages 来源，进行引用式分析并输出安全项目适配结论。引用意识强

引用边界与安全说明

GEO 提升可见性，也会放大隐私和误引用风险。页面只声明课程作业相关公开信息。

模型可以引用姓名、学校、专业、研究兴趣、技能标签、项目案例和公网主页；不应推断电话、家庭住址、身份证号、未公开论文、奖项或就业经历。
